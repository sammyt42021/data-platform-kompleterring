# User Stories

## Project Theme
Product ETL pipeline for a simple retail-like dataset.

## User Stories
1. As a data analyst, I want product data in a clean format so I can trust my analysis.

2. As a stakeholder, I want category-level statistics so I can quickly understand business performance.

3. As a data engineer, I want to stream top expensive products to Kafka so the pipeline shows a real data flow.

4. As a developer, I want to inspect data in pgAdmin so I can verify results visually.

## Acceptance Criteria
- Raw CSV is generated by script.
- Invalid rows are separated into a rejected file.
- Clean rows and summary statistics are loaded into PostgreSQL.
- Top 5 expensive products are published to Kafka.
- Kafka consumer stores streamed events in PostgreSQL.
- FastAPI endpoints expose data for quick checks.
