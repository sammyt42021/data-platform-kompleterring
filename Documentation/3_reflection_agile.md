# Kort Reflektion Kring Agilt Arbete

Jag arbetade i små inkrement: data generation -> transform -> databas -> streaming -> API.  
Det passade bra för den här typen av komplettering eftersom varje steg gick att köra och verifiera separat.

Det som fungerade bra:
- Jag definierade en enkel MVP tidigt.
- Jag delade upp arbetet i små script i stället för ett stort program.
- Jag dokumenterade löpande, vilket gjorde det lättare att hålla riktning.

Det som kunde göras bättre:
- Jag kunde ha skapat fler tydliga GitHub-issues från början.
- Jag kunde ha gjort mindre och mer frekventa PR:er per delmoment.
- Jag kunde ha haft en kort retrospektiv efter varje sprint.

Förbättringar till nästa projekt:
- Starta med en enkel Kanban-tavla.
- Skriva testfall tidigare i sprinten.
- Sätta tydlig tidsbox för dokumentation och reflektion efter större commit.
