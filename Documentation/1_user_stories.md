# User Stories

## Theme
I chose a product dataset to demonstrate a complete ETL flow in a manageable scope.

## Stories
1. As a data analyst, I need clean product data so I can trust the analysis.
2. As a teacher/reviewer, I need to see a clear ETL chain so the implementation can be followed step by step.
3. As a stakeholder, I need category-level statistics so I can quickly understand performance patterns.
4. As a data engineer, I need Kafka integration so the pipeline is not only batch-based.
5. As a developer, I need database visibility in pgAdmin so I can verify loaded results.

## Acceptance Criteria
- The raw CSV is generated by code.
- Invalid rows are kept and explained in a rejected file.
- Clean and summary data are loaded into PostgreSQL.
- Top expensive products can be published to Kafka.
- Kafka consumer writes events into PostgreSQL.
- API endpoints return useful checks for products and stats.
